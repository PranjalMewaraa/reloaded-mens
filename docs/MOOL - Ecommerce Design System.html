01 — Brief

A quiet system for everyday menswear,
built mobile-first for the Indian market.

MOOL. is the working brand name for this exercise — chosen for its short, ownable feel and Hindi root meaning origin. The system is Premium Minimalist + Soft UI: pure white canvas, cool-gray neutrals, deep charcoal type, one vibrant orange for action, sage green for contextual moments. Plus Jakarta Sans throughout. No 90° corners — extreme rounding sets the editorial tone. Rebuilt for ecommerce surfaces the reference doesn't cover: variant pickers, pincode serviceability, GST-inclusive pricing, OTP, WhatsApp-led in-store-only path.

Space, radius, motion.

Auto-layout & constraint conventions
  • Every frame uses auto-layout; nothing is absolutely positioned except phone bezel notch + image overlay badges.
  • Padding scale fixed to the 4px ladder (4/8/12/16/20/24/32/40/48/64/80/96). No off-grid numbers.
  • Default content padding · mobile: 20 horizontal, section gap 32. Desktop: 96 horizontal, section gap 80.
  • Resizing · screen frames are width=fixed (375 / 1440) with content set to "fill container". Inner cards default to "fill container · hug vertical".
  • Hit area · all interactive primitives have min height 44 mobile / 40 desktop. Tap targets <44 must wrap in a padded interactive frame.
  • Naming · components are Pattern / Element / Variant. Instances inside pages are renamed to their role (e.g. btn-add-to-bag).
  • Color & text styles are tokens — never local. Variants flip the style assignment, never the hex.
  • Two breakpoints only: 375 (mobile) and 1440 (desktop). Tablet derives from desktop with 64 horizontal padding.
